# Iteration 0.0.2 — 操作审计日志

## 范围

新增操作审计日志功能模块。在已有 AuditLog 数据模型基础上，实现后端自动记录和前端管理员专属只读审计日志页面。

## 来源

- **owner_event**: e000020 (2026-06-18T11:33:32Z)
- **request_id**: eval-20260618-003-audit-log
- **kind**: feature（新功能迭代）

## 需求（来自老板原话）

1. 记录所有用户关键操作（登录、登出、创建合同、修改合同状态、上传附件、删除附件、创建/编辑/禁用用户）
2. 后端自动记录操作人、操作类型、操作对象、操作时间、IP 地址
3. 新增管理员专属审计日志页面，支持按时间范围、操作类型、操作用户筛选
4. 审计日志只读，任何用户不可删除或修改
5. 日志列表分页展示，按时间倒序

## 业务模型变更

扩展现有 AuditLog 实体（已有字段 id, user_id, action, target_type, target_id, detail, created_at）：

- 新增字段：`ip_address`（操作来源 IP）
- action 枚举扩展：
  - `login`, `logout`（用户会话）
  - `contract_create`, `contract_update`, `contract_status_change`, `contract_delete`（合同操作）
  - `attachment_upload`, `attachment_delete`（附件操作）
  - `user_create`, `user_edit`, `user_disable`, `user_enable`（用户管理）

## 产品流程

1. 审计日志记录：后端中间件/AOP 在关键 API 调用时自动写入 AuditLog
2. 管理员登录 → 导航栏新增「审计日志」入口（仅 admin 可见）
3. 审计日志页面：
   - 筛选栏：时间范围（日期选择器）、操作类型（下拉多选）、操作用户（搜索选择）
   - 日志列表：表格展示（时间、操作人、操作类型、操作对象、IP 地址、详情）
   - 分页：每页 20 条，按时间倒序
   - 只读：无编辑/删除按钮
