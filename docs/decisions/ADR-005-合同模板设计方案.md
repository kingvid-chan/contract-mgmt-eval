# ADR-005: 合同模板设计方案 — 软删除 + {{变量}} 占位符

**状态**: 已采纳  
**日期**: 2026-06-18  
**迭代**: 0.0.3

## 背景

0.0.3 迭代需新增合同模板功能：管理员可创建和管理模板，用户创建合同时可从模板下拉选择快速填充。需选择模板删除策略和占位符格式。

## 方案对比

### 模板删除策略

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| **A: 软删除** | `is_deleted` 布尔字段，查询时过滤 | 数据可追溯、不影响已创建合同、可恢复 | 查询需额外过滤条件 |
| B: 硬删除 | 直接 DELETE FROM | 简单直接 | 已通过模板创建的合同丢失模板引用 |
| C: 归档表 | 删除时移动到 archive 表 | 主表干净、数据可追溯 | 复杂度高、引入额外表 |

### 占位符格式

| 方案 | 格式示例 | 优点 | 缺点 |
|------|---------|------|------|
| **A: `{{变量}}`** | `{{甲方}}` | 直观、常见模板引擎格式、不易误匹配 | 需处理 `{{` `}}` 字面量 |
| B: `${变量}` | `${甲方}` | 简洁 | 与 JS 模板字符串冲突、前端渲染需额外处理 |
| C: `【变量】` | `【甲方】` | 中文友好 | 与常见文档标记冲突 |

## 决策

### 删除策略：选择方案 A — 软删除

- 在 `contract_templates` 表新增 `is_deleted BOOLEAN DEFAULT FALSE` 字段
- 所有查询端点默认添加 `WHERE is_deleted = FALSE` 过滤条件
- 删除操作仅设置 `is_deleted = TRUE`，不物理删除行
- 理由：数据可追溯、不影响已通过模板创建的合同、实现简单

### 占位符格式：选择方案 A — `{{变量}}`

- 模板正文中使用 `{{变量名}}` 语法
- 创建合同时，系统将模板中的占位符替换为合同表单实际值
- 替换映射：`{{甲方}}` → `party_a_value`, `{{乙方}}` → `party_b_value`, `{{金额}}` → `amount`, `{{日期}}` → `sign_date`, `{{合同编号}}` → `contract_no`
- 理由：直观易理解、与主流模板引擎（Jinja2/Handlebars）一致、中文环境不产生歧义

## 影响

- 后端 `template_service.py`：查询时过滤 `is_deleted=False`
- 后端 `template_service.py`：`render_template_content()` 函数实现占位符替换
- 前端：模板列表不显示已删除模板
- 数据库：`contract_templates` 表新增 `is_deleted` 列
- 审计日志：模板删除记录 `action='template.delete'`
