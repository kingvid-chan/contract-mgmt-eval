# ADR-002: 审计日志记录方式 — FastAPI 依赖注入

**状态**: 已采纳  
**日期**: 2026-06-18  
**迭代**: 0.0.2

## 背景

需要自动记录所有关键 API 操作（登录、登出、创建/修改合同状态、上传/删除附件、创建/编辑/禁用用户）的审计日志，包含操作人、操作类型、操作对象、操作时间和客户端 IP 地址。

## 方案对比

| 方案 | 做法 | 优点 | 缺点 |
|------|------|------|------|
| A: ASGI 中间件 | `@app.middleware("http")` 拦截所有请求 | 完全自动、不遗漏 | 难以解析路由参数、难以获取操作对象 ID |
| **B: FastAPI 依赖注入** | 创建 `AuditLogger` Depends 在路由层注入 | 精准控制端点、可访问 Request 对象、与现有 `get_current_user` 模式一致 | 每个端点需手动注入 |
| C: 仅扩展 Service 层 | 保持现有 `log_action()` 调用 | 改动最小 | 不够「自动」，依赖开发者记得调用 |

## 决策

选择 **方案 B — FastAPI 依赖注入**。

`AuditLogger` 类仅依赖 `Request`（自动注入），不耦合 `get_current_user`。IP 提取和用户认证独立注入：

```python
class AuditLogger:
    def __init__(self, request: Request):
        self._request = request
    
    @property
    def ip_address(self) -> str:
        ...
```

路由层同时注入 `get_current_user`（或 `require_admin`）和 `AuditLogger`，将 `audit.ip_address` 传递给 service 层。

## 影响

- 新增 `backend/app/middleware/audit.py`
- 所有需要审计的路由函数新增 `audit: AuditLogger = Depends()` 参数
- Service 函数新增 `ip_address: str | None = None` 可选参数
- `log_action()` 工具函数新增 `ip_address` 参数
